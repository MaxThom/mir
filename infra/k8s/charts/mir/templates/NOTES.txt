1. Get the application URL by running these commands:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "mir.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if contains "LoadBalancer" .Values.service.type }}
     NOTE: It may take a few minutes for the LoadBalancer IP to be available.
           You can watch its status by running 'kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "mir.fullname" . }}'
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "mir.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo http://$SERVICE_IP:{{ .Values.service.port }}
{{- else if contains "ClusterIP" .Values.service.type }}
  export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/name={{ include "mir.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}")
  export CONTAINER_PORT=$(kubectl get pod --namespace {{ .Release.Namespace }} $POD_NAME -o jsonpath="{.spec.containers[0].ports[0].containerPort}")
  echo "Visit http://127.0.0.1:8080 to use your application"
  kubectl --namespace {{ .Release.Namespace }} port-forward $POD_NAME 8080:$CONTAINER_PORT
{{- end }}

2. NATS Configuration:
{{- if .Values.nats.enabled }}
   NATS has been deployed as a subchart with the following configuration:
   - Service: {{ include "mir.fullname" . }}-nats:4222
   - JetStream: {{ .Values.nats.config.jetstream.enabled }}
   - Monitor Port: {{ .Values.nats.config.monitor.port }}

   To access NATS monitoring:
   kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "mir.fullname" . }}-nats 8222:8222
   Then visit http://localhost:8222
{{- else }}
   NATS subchart is disabled. Mir is configured to use external NATS at: {{ .Values.config.mir.url }}
   Make sure your external NATS instance is accessible.
{{- end }}

3. To verify Mir is connected to NATS:
   kubectl --namespace {{ .Release.Namespace }} logs -l "app.kubernetes.io/name={{ include "mir.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" | grep -i nats

4. SurrealDB Configuration:
{{- if .Values.surrealdb.enabled }}
   SurrealDB has been deployed as a subchart with the following configuration:
   - Service: {{ include "mir.fullname" . }}-surrealdb:8000
   - Storage Backend: File-based (surrealkv) with {{ .Values.surrealdb.persistence.size }} persistent volume
   - Authentication: {{ if .Values.surrealdb.surrealdb.auth }}Enabled{{ else }}Disabled (enable after initial setup){{ end }}

   To access SurrealDB Studio:
   kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "mir.fullname" . }}-surrealdb 8000:8000
   Then visit http://localhost:8000

   {{- if .Values.surrealdb.ingress.enabled }}
   Or access via ingress at: http://{{ .Values.surrealdb.ingress.host }}
   {{- end }}
{{- else }}
   SurrealDB subchart is disabled. Mir is configured to use external SurrealDB at: {{ .Values.config.surreal.url }}
   Make sure your external SurrealDB instance is accessible.
{{- end }}

5. To verify Mir is connected to SurrealDB:
   kubectl --namespace {{ .Release.Namespace }} logs -l "app.kubernetes.io/name={{ include "mir.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" | grep -i surreal
