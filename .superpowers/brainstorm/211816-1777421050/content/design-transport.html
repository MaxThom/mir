Section 2 of 4 — HTTP/SSE wire protocol & library

MCP wire format (HTTP/SSE)

1. AI client connects
GET /mcp/sse
← event: endpoint
← data: /mcp/message?sid=abc
2. AI calls a tool
POST /mcp/message?sid=abc
{"method":"tools/call",
 "params":{"name":"list_devices"}}
← event: message
← data: {"result": [...]}
3. AI lists tools
POST /mcp/message?sid=abc
{"method":"tools/list"}
← event: message
← data: {"tools": [...]}

Library recommendation: mark3labs/mcp-go

Handles JSON-RPC 2.0, session management, SSE plumbing, and tool schema generation automatically. We just register tools and their handlers.

s := server.NewMCPServer("Echo", "1.0.0")

s.AddTool(mcp.NewTool("list_devices",
  mcp.WithDescription("List Mir devices"),
  mcp.WithString("namespace", mcp.Optional()),
), e.handleListDevices)

// Plug into our mux:
sseServer := server.NewSSEServer(s)
mux.Handle("/mcp/", sseServer)
