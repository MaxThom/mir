Echo — Design: Tool Definitions

Section 3 of 4 — All tools with typed input structs

Operational tools — talk to Mir via NATS

list_devices(namespace?, name?) → []Device
get_device(name, namespace) → Device (full digital twin)
create_device(name, namespace, schema?) → Device
delete_device(name, namespace) → ok
send_command(name, namespace, command, payload_json) → response
update_config(name, namespace, config, payload_json) → ok
get_telemetry(name, namespace, field?, limit?) → []TlmPoint
list_events(namespace?, type?, limit?) → []Event

Developer assistant tools — docs + CLI introspection

query_docs(query) → relevant book/src sections as text
explain_cli_command(command) → runs `mir <command> --help`, returns output
list_cli_commands() → runs `mir --help`, returns command tree
scaffold_device_project(module_name) → runs `mir tools generate device-template`
generate_schema(device_name) → runs `mir tools generate mir-schema`

Example: typed structs in Go

// tools_operational.go
type SendCommandInput struct {
  Name string `json:"name" jsonschema:"required,Device name"`
  Namespace string `json:"namespace" jsonschema:"required,Namespace"`
  Command string `json:"command" jsonschema:"required,Command name"`
  Payload string `json:"payload" jsonschema:"JSON payload string"`
}

// tools_devassist.go
type QueryDocsInput struct {
  Query string `json:"query" jsonschema:"required,Natural language question about Mir"`
}
