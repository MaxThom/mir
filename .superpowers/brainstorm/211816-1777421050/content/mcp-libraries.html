Go MCP Server Libraries

Two real contenders — pick one. The third is out.

A

mark3labs/mcp-go  ⭐ Ecosystem standard

8,644 ★v0.49.0Last commit: 2026-04-21OpenIssues: 34

De-facto standard in the MCP Go ecosystem. Builder-pattern API, weekly releases, OAuth already shipping.

s := server.NewMCPServer("Echo", "1.0.0")
s.AddTool(mcp.NewTool("list_devices",
  mcp.WithDescription("List Mir devices"),
  mcp.WithString("namespace", mcp.Optional()),
), handler)
sseServer := server.NewSSEServer(s)
mux.Handle("/mcp/", sseServer)

Pros

  • Most community examples & docs
  • Battle-tested HTTP/SSE transport
  • Middleware, auth, test utilities
  • Integrates as http.Handler — fits our mux exactly

Cons

  • Not the "official" SDK
  • v0.x — API can still change
B

modelcontextprotocol/go-sdk  🏢 Official SDK

4,448 ★v1.5.0Last commit: 2026-04-28OpenSSF ✓

Maintained by Anthropic + Google. Typed handler pattern, auto-schema from Go structs. v1 — stable API.

s := mcp.NewServer("Echo", "1.0.0", nil)
mcp.AddTool(s, &mcp.Tool{
  Name: "list_devices",
}, typedHandler)
h := server.NewSSEHandler(s)
mux.Handle("/mcp/", h)

Pros

  • Official — long-term winner
  • v1.x — stable, no breaking changes
  • OpenSSF security review
  • Auto-schema from Go structs

Cons

  • Fewer community examples
  • Newer — less battle-tested at scale
  • Client OAuth still experimental
Eliminated

metoro-io/mcp-golang — last commit Sep 2025, no streamable-HTTP, minimal community. Skip it.

My recommendation: A (mark3labs) for v1 — more examples to reference, battle-tested SSE transport, integrates cleanly. Switch to official SDK once it matures.
