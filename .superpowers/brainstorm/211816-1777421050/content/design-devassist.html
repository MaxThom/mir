Section 4 of 4 — Embedded docs + live CLI introspection

Embedded docs (docs_embed.go)

All book/src/**/*.md files are compiled into the binary at build time — same pattern as the web UI embed.

//go:embed book/src
var bookFS embed.FS

// query_docs walks bookFS, scores each .md file
// by keyword overlap with the query, returns top-3 sections

No external service, no LLM call — pure string matching. Fast, zero latency, works offline.

Live CLI introspection

explain_cli_command and list_cli_commands exec into the mir binary at runtime:

// explain_cli_command("dev cfg send")
exec.Command("mir", "dev", "cfg", "send", "--help")
// → returns live --help output, always current

// list_cli_commands()
exec.Command("mir", "--help")
// → returns command tree

Always reflects the actual installed version — no stale docs problem.

How it comes together for the AI

Developer question

"How do I send a config update to a device?"

AI calls query_docs("send config device") → gets the relevant book section → synthesizes an answer with code examples.

CLI question

"What flags does mir dev cfg send take?"

AI calls explain_cli_command("dev cfg send") → gets live --help output → returns it directly.

Scaffold request

"Bootstrap a new sensor device project"

AI calls scaffold_device_project("my-sensor") → runs mir tools generate device-template my-sensor → returns generated files.

Fleet operation

"Reboot all devices in prod that are offline"

AI calls list_devices("prod"), filters by status, loops send_command(..., "reboot", "") for each.
